Step 3 — EAD-Weighted Overall RAG

The 1y and 2y horizon RAGs are combined:

Weighted Averaging
  1. Score each horizon RAG: Green = 1 Amber = 2 Red = 3
  2. Weight by EAD share for each horizon (ead_share column)
  3. Weighted average = Σ(score × weight) / Σ(weight)
  4. Round (half-down) → map back to RAG

Example: 1y RAG = Green (1) at 60% EAD, 2y RAG = Amber (2) at 40% EAD
Weighted avg = (1 × 0.6 + 2 × 0.4) / 1.0 = 1.4 → rounds to 1 → Green

5. Multi-Model Aggregation — Open Decision

When the user selects "All" segments and multiple models, the dashboard must combine metrics across models. Three approaches are available:

How should cross-model metrics be combined?
Option A: Row Pooling (Current Facility-Level Behavior)

Pool all facility-level rows from all models and compute metrics on the combined set. This is what the dashboard does today with facility-level data. Not applicable for pre-aggregated data since individual facility rows don't exist.

Option B: EAD-Weighted Average (Recommended)

Compute each model's metrics independently, then combine via EAD-weighted average. Most accurate for pre-aggregated data. Each model's pre-computed metric retains its statistical validity.

Formula: Combined Metric = Σ(Modeli metric × Modeli EAD) / Σ(Modeli EAD)

Option C: Show Per-Model Only

When multiple models are selected, show each model's metrics separately — no cross-model aggregation. Simplest approach, avoids combining metrics that may not be statistically meaningful to combine.

6. Segment Filtering Behavior

The segment column in the aggregated sheet may contain comma-separated values (e.g. "Defensive, O&M") for models that cover multiple segments.

Filter Behavior
User SelectionBehavior
AllAll observations pass — no segment filtering
CyclicalOnly observations with segment = "Cyclical" pass
DefensiveOnly observations tagged "Defensive" pass (model with "Defensive, O&M" included)
O&MOnly observations tagged "O&M" pass

The loader splits multi-segment values and creates separate observation sets per segment. EAD is divided equally across segments.
